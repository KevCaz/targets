3  Help

This chapter explains how to ask for assistance. To ensure a constructive dialogue and a positive outcome, please read the entire page before reaching out.

3.1 Is it really about targets?

The targets package itself is rarely the cause of an issue in a targets pipeline. Most problems come from the user-defined R code, R commands, and R functions that the pipeline calls, as well as other R packages on your system. Before posting, please attempt to troubleshoot and figure out if targets is really the cause, using the tips in the debugging chapter as needed. If it turns out to be a non-targets issue, then please ask your question in a non-targets forum and rewrite the question accordingly.

3.2 Search existing posts

Please check if your question already has an answer. The bottom of this page links to searchable forums with hundreds of existing discussions. If a relevant thread already exists, please read it first. Then if necessary, please comment there instead of starting a new post.

3.3 Post the complete context

Computer code is extremely strict. In order to run properly, it needs to be exactly correct, down to the last detail. That means for every bug you discover, there are hundreds of strange and unguessable potential causes. The culprit could be a typo buried deep in the targets codebase, an incorrect hidden assumption lost in translation about using targets, an obscure bug in an obscure R package dependency, a mistake in one of your R functions, or some other mysterious or unexpected reason. In addition, you may be the first user to experience the problem. So when you ask for help, it is not enough to state the error and ask what might be the solution. The true answer is too surprising to predict, even for the most experienced experts of R and targets.

To receive useful feedback, please provide the complete context of the problem, including a reprex. The purpose of a reprex, or reproducible example1, is to eliminate the knowledge gaps and hidden assumptions where bugs hide. A reprex is a sample of complete, self-contained, runnable code that fully emulates and reproduces the problem. The code should look clean, be as short and concise as possible, run in as few seconds as possible, and contain only the details most relevant to troubleshooting. You can embed the code inline in your question, or you can upload it to a public repository and post the link. Regardless, please expect that anyone trying to help will read all the code and run the enclosed _targets.R file on their own private computer. Please make this empirical process as quick and easy as possible for the people who volunteer their valuable time and energy to respond to you.

3.4 Code of conduct

All developers, users, and other discussants must follow the code of conduct. Please read it before commenting or posting.

3.5 Sensitive information

The forums linked below are open to the public, and anyone with an internet connection can read them. Please do not post any company confidential information, personally identifiable information, private research data, or any other sensitive information.

3.6 R community

You can reach out to the R community at these forums.

  • Stack Overflow
  • Posit Community
  • rOpenSci Discuss
  • Mastodon (#rstats hashtag)
  • R subreddit

3.7 Contact the maintainer

To contact the maintainer directly, please post to the relevant public GitHub Discussions page.2 Examples:

  • targets: https://github.com/ropensci/targets/discussions
  • tarchetypes: https://github.com/ropensci/tarchetypes/discussions
  • gittargets: https://github.com/ropensci/gittargets/discussions
  • jagstargets: https://github.com/ropensci/jagstargets/discussions
  • stantargets: https://github.com/ropensci/stantargets/discussions

Anyone can find and link to a public GitHub discussion, and the ability to search and cross-reference helps the users and the maintainer. For this reason, please do not send private emails, instant messages, or mentions on social media.

3.8 Out of office

When the maintainer is out of office, the GitHub repositories above will enter vacation mode. During these times, Github interactions are temporarily limited, which means users cannot open or comment on issues or discussions. Each repository will have an open issue like this one to explain vacation mode and show the date that the maintainer will return and re-enable interactions.3

3.9 Thanks

Thank you for reading this chapter, thank you for taking an active role in troubleshooting, and thank you for respecting the time and effort it takes to help users.
